DDoS Attack Prevention & Mitigation using Stratified Architecture

A DDoS (Distributed Denial of Service) attack is an attempt to make an online service unavailable by overwhelming it with traffic from multiple sources. These attacks are typically carried out using a botnet, which is a network of infected computers or devices that can be controlled remotely to send traffic to a target server or network.

Tech Stack

  • Databases: XAMPP, SQL
  • Frontend: Flask, react.js
  • Middleware: node.js
  • Backend: tf.js, express.js

Procedure

Our system consists of 2 tables namely, Legal MAC address table and Intruders MAC address table. If a device’s MAC address is present in Intruders MAC address table, its connection is blocked. If a device’s MAC address is not present in either of table, the MAC address is added to Legal MAC address table and connection is forwarded to the next authentication layer i.e. puzzle layer. If a device’s MAC address is present in Legal MAC address table, it is checked how many times that device tried to access the website. If the count is found to be greater than 10 in last 5 minutes then the connection is blocked.

User has to solve a “draw & shape” (instead of “dot and shape”) and “integer arithmetic” puzzle. Draw & Shape: user has to draw on given canvas and the embedded json model (deployed using tfjs) will inference the same. If the inference and asked shape is same (triangle/circle/square), user passes the puzzle. Model is embedded in the server, rather than the end device’s local storage. After verification, the user is redirected to “integer arithmetic layer”. User has to solve a random arithmetic question, and if solved correctly, user is redirected to home page.
